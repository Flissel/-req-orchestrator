# Bad Requirements Examples - Testing Auto-Validation

This file contains intentionally poorly-written requirements to test the auto-validation system. Each requirement violates one or more quality criteria.

## Vague Requirements (Clarity Issues)

REQ-001: Die App muss schnell sein

REQ-002: Das System soll benutzerfreundlich sein

REQ-003: Die Software muss gut funktionieren

## Non-Testable Requirements (Testability Issues)

REQ-004: Die Anwendung sollte schön aussehen

REQ-005: Das Interface muss intuitiv sein

REQ-006: Die App soll moderne Technologien verwenden

## Non-Measurable Requirements (Measurability Issues)

REQ-007: Die Ladezeit soll kurz sein

REQ-008: Der Speicherverbrauch muss niedrig bleiben

REQ-009: Die Verfügbarkeit soll hoch sein

## Non-Atomic Requirements (Atomic Violation)

REQ-010: Das System muss Daten speichern, anzeigen, bearbeiten und löschen können

REQ-011: Die App soll auf iOS und Android laufen, Offline-Modus haben und Push-Benachrichtigungen senden

REQ-012: Das Backend muss API bereitstellen, Authentifizierung handhaben, Logs schreiben und Fehler behandeln

## Too Verbose Requirements (Concise Violation)

REQ-013: Als ein Benutzer der Anwendung möchte ich in der Lage sein, mich mit meinem Benutzernamen und meinem Passwort anzumelden, wobei das System überprüfen soll, ob meine Anmeldedaten korrekt sind, und wenn ja, dann soll ich Zugang zum System erhalten, und wenn nein, dann soll eine Fehlermeldung angezeigt werden, die mir mitteilt, dass meine Anmeldedaten falsch sind

## Ambiguous Requirements (Unambiguous Violation)

REQ-014: Das System speichert die Daten

REQ-015: Der Nutzer kann Einstellungen ändern

REQ-016: Die App zeigt Informationen an

## Inconsistent Language (Mixed German/English)

REQ-017: User shall be able to login

REQ-018: Das System muss den login process durchführen

## Missing Template Adherence

REQ-019: Schnelle Antwortzeit

REQ-020: Export als PDF

## Design-Dependent Requirements

REQ-021: Die App muss React verwenden

REQ-022: Die Datenbank soll PostgreSQL sein

REQ-023: Der Button muss blau sein und rechts oben platziert werden

## Purpose-Dependent Requirements

REQ-024: Das System muss für das Marketing-Team Reports generieren

REQ-025: Die App soll unsere Firma als modern darstellen
